Getting Started

This document will make your life a little easier when using the weather-api product. Please read the introduction. And if you want to use the endpoint, scroll down further. It will probably come out. If an error code comes out, of course, it is highly likely that you wrote it down incorrectly, but I wrote it down below, so take a look at it.

Introduction

Weather-api allows you to get weather data for weather station coordinates in South Korea with a simple programmatic method. If the request is successful, the response is sent back in JSON or CSV object format.

Note: Current version is v1

API Endpoint

  • Endpoint BaseURL: http://gausslab-hq.iptime.org/weather-api/v1

Authentication

Weather-api uses an API key to allow access to the API.

It expects to be included with headers like:

  • header = { ‘Authorization’: ‘example-api-key’ }

Note: replace example-api-key with your personal API key

Endpoints

CSV

  • Endpoint: http://gausslab-hq.iptime.org/weather-api/v1/csv

  • POST
    • : upload CSV File for weather data
    • - @Requestbody : File
      • (File : weather data CSV)
      - Response : List<long>
      • (Long : Generator ID)

  • POST {CSV Endpoint}/station
    • : upload CSV File for stationGenerator data
    • - @Param 1 : stationFile
      • (File : Station Data CSV)
      - @Param 2 : gisFile
      • (File : Gis Data CSV)
      - Response : List<long>
      • (Long: Generator ID)

  • GET {CSV Endpoint}/history
    • : get CSV File content
    • - @Param 1 : generatorId
      • (generatorId : Long)
      - @Param 2 : startDate
      • (start Date : String, Format : ISO Formatted Timestamp -> 2022-03-08T15:00:00)
      - @Param 3 : endDate
      • (end Date : String, Format : ISO Formatted Timestamp -> 2022-03-10T15:00:00)
      - Response : String
      • (String : CSV File content)

    StationGenerator

    • Endpoint: http://gausslab-hq.iptime.org/weather-api/v1/generator

  • GET
    • : get all generators
    • - Response : List<GeneratorDto>
      • (GeneratorDto - station information + generator information
      • : id, generatorName, stationId, stationRegion, stationAddress, latitude, longitude, type, capacity)

  • GET {StationGenerator Endpoint}/{id}
    • : get generator information by generator ID
    • - @PathVariable : {id}
      • ({id} : Long, generatorId)
      - Response : GeneratorDto
      • (GeneratorDto - station information + generator infofmation
      • : id, generatorName, stationId, stationRegion, stationAddress, latitude, longitude, type, capacity)

    Weather

    • Endpoint: http://gausslab-hq.iptime.org/weather-api/v1/history

  • GET
    • : get weather data for generator Id and time Range
    • - @Param 1 : generatorId
      • (generatorId : Long)
      - @Param 2 : startDate
      • (start Date : String, Format : ISO Formatted Timestamp -> 2022-03-08T15:00:00)
      - @Param 3 : endDate
      • (end Date : String, Format : ISO Formatted Timestamp -> 2022-03-10T15:00:00)
      - Response : List<WeatherData>
      • (WeatherData
      • : id, generatorId, timestamp, timestampUtc, timestampLocal, partOfDay, precipitation. cloudCoverage, snow, visibility, temperature, apparentTemperature, relativeHumidity, dewPoint, windSpeed, windDirection, pressure, seaLevelPressure, uvIndex, solarElevationAngle, solarRadiation, solarAzimuthAngle, globalHorizontalSolarIrradiance, directHorizontalSolarIrradiance, directNormalSolarIrradiance)

    Error Codes

    If an error code appears as the response value, take a look below.

    400 : Bad Request

    The request sent by the user is not valid.

    401 : Unauthorized

    The user's API key you currently have is incorrect.

    429 : Too many Requests

    There are currently too many requests. Please try again in a few minutes.

    500 : Internal Server Error

    There is a problem with our server.

    503 : Service Unavailable

    Currently we are temporarily offline for maintenance.
